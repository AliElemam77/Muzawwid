# محوّل الشيتات إلى صيغة سلة

أداة تعمل بالكامل داخل المتصفح لتحويل **أي** ملف منتجات (`.xlsx` أو `.xls` أو `.csv`) بأي أعمدة إلى **ملف استيراد منتجات سلة** (`.xlsx`) جاهز للرفع على:

<div dir="ltr">

```
https://s.salla.sa/import/products
```

</div>

لا يوجد خادم، ولا رفع لأي ملف، ولا مفاتيح API — كل المعالجة تتم في متصفحك.

## التشغيل

<div dir="ltr">

```bash
pnpm install
pnpm dev       # تشغيل بيئة التطوير
pnpm build     # فحص الأنواع + بناء الإنتاج
pnpm test      # تشغيل اختبارات كاتب صيغة سلة
```

</div>

ثم افتح الرابط الذي يظهره Vite في المتصفح.

## طريقة الاستخدام (٣ خطوات)

1. **رفع الملف** — اسحب ملفك أو اختره. إذا كان الملف يحتوي أكثر من ورقة، اختر الورقة المطلوبة. تظهر معاينة لأول ١٠ صفوف من أعمدتك.
2. **مطابقة الأعمدة** — لكل حقل في سلة اختر أحد ثلاثة:
   - **عمود مصدر** من ملفك، أو
   - **قيمة ثابتة** تُطبَّق على كل الصفوف، أو
   - **بدون** (يُترك فارغًا).
   تُقترح المطابقات تلقائيًا بمطابقة أسماء الأعمدة (عربية وإنجليزية) مع حقول سلة، وتبقى كلها قابلة للتعديل.
3. **التصدير** — تُعرض معاينة حيّة للمخرجات مع تقرير تحقق، ثم يُنزّل الملف باسم `salla-import.xlsx`.

## كيف تعمل المطابقة

- **دمج الصور**: اختر عدة أعمدة صور تُدمج روابطها (بدون تكرار، مفصولة بفاصلة) في عمود «صورة المنتج».
- **توليد SKU**: من عمود، أو استخراج من رابط عبر النمط `‎/p(\d+)‎` مع بادئة، أو ترقيم تلقائي. رمز الخيار = `{رمز المنتج}-{قيمة الخيار}`.
- **الخيارات (المتغيرات)**: حدّد حتى ٣ أعمدة كخيارات (نص / لون / صورة). كل قيمة تتوسّع إلى صف «خيار» أسفل المنتج الأب، ويرث سعر الأب ما لم يُربط عمود سعر خاص بالمتغير.
- **القيم الافتراضية**: قيم ثابتة قابلة للتعديل تُملأ في كل صف فارغ — أهمها **الوزن = 1** و**وحدة الوزن = kg** (مطلوبان من سلة).
- **تنظيف الأسعار**: تُزال رموز العملات والفواصل، وتُعامل `-` أو الفراغ كقيمة فارغة.

## آلية المنتج / الخيار

العمود الأول «النوع» يحدّد نوع الصف:

- **`منتج`** = صف المنتج الأب (يحمل الاسم والصور والسعر والرمز والماركة…).
- **`خيار`** = صف متغيّر أسفل الأب مباشرة (يحمل قيمته الخاصة + سعره/رمزه إن وُجد).

على صف المنتج تُكتب أسماء الخيارات وأنواعها فقط، ويوضع في خانة `[n] القيمة` نص توضيحي ثابت، بينما تُكتب القيم الفعلية في صفوف «خيار».

## قاعدة «ورقة واحدة فقط» ⚠️

سلة ترفض الملف إذا احتوى على أكثر من ورقة (رسالة الخطأ: **«يرجى حذف المصنفات الإضافية»**). لذلك يُنتج هذا المحوّل **ورقة واحدة فقط** باسم:

<div dir="ltr">

```
Salla Products Template Sheet
```

</div>

مع صفّي ترويسة إلزاميين: صف تسمية القسم (`بيانات المنتج`) ثم صف الأعمدة الأربعين بالترتيب الدقيق. لا تُضِف أي ورقة أخرى قبل الرفع.

## الحقول المطلوبة (تمنع التصدير إذا كانت فارغة)

- **أسم المنتج**
- **سعر المنتج** (على الصف الحامل للسعر)
- **الوزن** — مطلوب على صفوف المنتج والخيار معًا (خطأ سلة: «حقل الوزن مطلوب»)، ويُملأ تلقائيًا بـ `1` / `kg`.
- **ورقة واحدة فقط** في الملف.

كما ينبّهك التحقق إلى تكرار الرموز (SKU) وصفوف الخيار بلا منتج أب، ويحذّرك (دون منع) من غياب الصور أو التصنيف أو الماركة.

## القوالب (Presets)

يمكن حفظ إعدادات المطابقة باسم في `localStorage` وإعادة تحميلها لاحقًا للملفات المشابهة.

## البنية التقنية

React + Vite + TypeScript، وتنسيق كامل بـ Tailwind CSS، وقراءة/كتابة الجداول عبر SheetJS (`xlsx`). واجهة عربية RTL بخط Tajawal. منطق التحويل في `src/lib/` والواجهة في `src/components/`.
